# K 线技术分析

## 目标

对股票、指数、ETF 或基金的 K 线走势做轻量技术分析，并输出可用于 K 线图画线的点位。

用户常见说法：

- “分析一下这只票 K 线”
- “帮我看下 K 线走势”
- “给我支撑压力和画线点位”
- “预测明天 K 线怎么画”
- “结合日本蜡烛图、趋势和量价分析”

## 数据来源

优先级：

1. 优先使用用户提供的 K 线数据、截图、表格、CSV、Excel 或网页链接。
2. 用户没有提供数据时，优先查询本项目 server 已有接口。
3. server 未启动、接口不可用或数据不完整时，再通过可靠公开行情源查询；如当前环境需要联网，按工具权限规则执行。
4. 不要凭模型记忆编造最新行情、价格和日期。

## Server 接口

股票、指数类 K 线：

```bash
curl "http://localhost:8000/api/basic/kline?code=<CODE>&period=daily&startDate=<YYYYMMDD>&endDate=<YYYYMMDD>&adjust=qfq"
```

返回核心字段：

```text
date, open, close, high, low, amount, amplitude, change_pct, change
```

ETF 基金 K 线：

```bash
curl "http://localhost:8000/api/basic/fund/kline?code=<ETF_CODE>&limit=120"
```

返回核心字段：

```text
date, open, close, high, low, volume, amount, amplitude, changePercent, changeAmount, turnoverRate
```

使用建议：

- 默认取最近 60 到 120 根日 K 线。
- 如果用户没给起止日期，股票/指数用近 4 到 6 个月区间；ETF 用 `limit=120`。
- 用户明确要周线或月线时，`period` 改为 `weekly` 或 `monthly`。

## 分析方法

保持轻量，不要写成复杂量化报告。重点看：

- 日本蜡烛图：十字星、锤头线、倒锤头、吞没、孕线、长上影、长下影、乌云盖顶、刺透、早晨之星、黄昏之星等。
- 趋势：最近高低点是否抬高或降低，是否处于上升、下降、震荡、反弹或回踩。
- 均线辅助：可计算或估算 MA5、MA10、MA20；只作为趋势辅助，不要堆太多指标。
- 支撑压力：用近期明显低点、高点、跳空缺口、长影线位置、平台密集成交区和整数关口。
- 量价关系：上涨放量、下跌缩量、放量滞涨、缩量回踩、放量破位、长下影承接等。
- 风险信号：高位长上影、放量阴线、跌破平台、反抽不过前高、缩量反弹。

## 点位计算口径

支撑位：

- 最近 20 到 60 根 K 线里的明显低点。
- 长下影最低点和收回位置。
- 前平台整理区下沿。
- 向上突破后的回踩确认位。

压力位：

- 最近 20 到 60 根 K 线里的明显高点。
- 长上影最高点和实体上沿。
- 前平台整理区上沿。
- 跌破后的反抽确认位。

趋势线：

- 上升趋势线使用两个或三个逐步抬高的低点。
- 下降趋势线使用两个或三个逐步降低的高点。
- 箱体线使用横向平台的上沿和下沿。

预测明日 K 线：

- 以最近收盘价、近 5 日波动、关键支撑压力和当日量价行为推演。
- 输出大致 `open`、`high`、`low`、`close`，保留与标的价格匹配的小数位。
- 预测是画图用的技术推演，不要表述成确定性结果。

## 输出格式

用中文输出，尽量直接。必须包含以下块：

```text
标的：<名称或代码>
数据区间：<开始日期> 至 <结束日期>

K 线形态：
<蜡烛图形态和短线含义>

趋势判断：
<上升/下降/震荡/反弹/回踩，以及原因>

支撑压力：
- 支撑 1：<价格>，原因：<低点/平台/长下影/均线等>
- 支撑 2：<价格>，原因：...
- 压力 1：<价格>，原因：...
- 压力 2：<价格>，原因：...

量价关系：
<成交量或成交额与价格行为的关系>

画线点位：
- 上升趋势线：(<日期>, <价格>) -> (<日期>, <价格>)
- 下降趋势线：(<日期>, <价格>) -> (<日期>, <价格>)
- 箱体上沿：<价格>
- 箱体下沿：<价格>
- 关键水平线：<价格列表>

明日预测 K 线：
open: <价格>
high: <价格>
low: <价格>
close: <价格>
形态倾向：<小阳/小阴/十字星/冲高回落/探底回升等>

交易推演：
<突破、回踩、破位三个情景，不要给绝对化建议>
```

输出要求：

- 点位必须能直接拿去画线，尽量给日期和价格。
- 如果数据不足以画趋势线，要明确说“趋势线证据不足”，但仍可给水平支撑压力。
- 不要给“必涨”“必跌”这类确定性判断。
- 不要长篇解释技术指标定义，直接用到分析里。
- 如果用户要求把结果写入项目文件，再按用户指定文件修改；普通分析请求只输出文字，不创建文件。
